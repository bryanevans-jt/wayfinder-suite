## Session 1 — Basic Operations (75 min)

**Goal:** ES can sign in, manage a caseload, log contacts, track applications, message participants, and submit a timesheet week.

### 1A — Sign in & orientation (15 min)

**Talking points:**

- Three apps, one mission: **Wayfinder Pro** (your daily work), **Wayfinder** (participants), **Joshua Tree Reports** (official PDFs → Drive). Same people, different URLs.
- Sign in at **wayfinder-pro.thejoshuatree.org** with `@thejoshuatree.org` — magic link, passkey, or Google. No passwords stored here.
- **Add to Home Screen** on phone — demo iPhone + Android briefly.
- Joshua Tree Reports uses a **separate sign-in** on its own URL — cover in Session 3, not now.
- Support: sidebar → **Help**; errors show **WF-** codes → Bryan Evans.

**Hands-on:** Everyone signs in to Wayfinder Pro and lands on **Clients**.

**Manual ref:** [Getting signed in](../shared/getting-signed-in.md) · ES Manual §1–2

---

### 1B — Clients & casework (35 min)

**Talking points:**

- **Clients** (`/dashboard/clients`) — your home base. Attention sort puts follow-ups at the top.
- Open a client profile — five areas ES touches daily:
  1. **Contact log** — log every call, visit, email. Counselors see **Notes** only; **Internal Notes** stay on the ES side. *This is P0 — it feeds counselor timelines and SE Monthly prefill.*
  2. **Applications** — pipeline: Applied → Interview → Offer → **Hired**.
  3. **Job start date** — set at hire; triggers 30/60/90-day counselor notifications.
  4. **Messages** — participant thread (detail in 1C).
  5. **Formal reports** — submission history (detail in Session 3).
- Use suggested **contact chips** when they match the activity.

**Hands-on (workbook Exercises 1–2):**

- Log a contact with public notes and an internal note.
- Add or update an application; set status toward Hired if using a test client.

**Manual ref:** ES Manual §3

---

### 1C — Messages & timesheet (15 min)

**Talking points:**

- **Messages** (`/dashboard/messages`) — reply to participants from Wayfinder (client app).
- **48 business-hour SLA** — respond promptly; supervisors coach from a read-only view; notifications flag overdue threads.
- **Timesheet** (`/dashboard/timesheet`) — log billable time by client and service date; **submit the week** when complete. Supervisors approve (covered with supervisors pre-conference).

**Hands-on (workbook Exercise 3):** Send a test message to a demo participant if available; enter one timesheet line.

**Manual ref:** ES Manual §4–5

---

### 1D — App pointers & participant setup (10 min)

**Talking points:**

- **Our Team** (sidebar) — two items only (values were covered in the culture session):
  - **Core Four** (`/dashboard/core-four`) — in-app reference for Strive for Excellence, Work with Confidence, Practice Discipline, Lead with Love. Open it when you want the scripture-backed summaries on the job.
  - **Share a Moment** (`/dashboard/share-moments`) — upload field photos (first day on the job, mock interviews, team wins). Add a short note; submission **emails leadership** with your photos attached. JTSG team members only — not counselors.
- **Client onboarding (2-minute checklist):** confirm email on client record → `wayfinder.thejoshuatree.org` → magic link together → tour Success path, Messages, Applications → test message received. Script: [Client setup guide](../scripts/client-setup-guide-for-es.md).
- **Counselors:** agency email only — ES coordinates with admin to get them provisioned; script in manual §12. Do not demo full counselor setup here.

**Manual ref:** ES Manual §10–11

## Session 2 — Community Partners Network (45 min)

**Goal:** ES can find employers on the map, share the join link, and connect outreach to casework.

### 2A — Map & directory (15 min)

**Talking points:**

- **Community Partners** (`/dashboard/community-partners`) — employer network for GA (and TN offices as you grow the map).
- **Map** — geographic view; **directory** — search and filter.
- Link employers to **applications** from client profiles as you place participants.
- Public join URL: generate tokenized invite from the app when possible; base path `wayfinder-pro.thejoshuatree.org/community-partners/join`.

**Hands-on:** Find an employer on the map; open directory entry.

**Manual ref:** ES Manual §7 · [Community Partners Field Guide](../shared/community-partners-field-guide.md)

---

### 2B — Employer outreach (25 min)

**Talking points:**

- Mindset: employers usually lack **information**, not empathy. Acknowledge, then pivot to Supported Employment + on-site coaching.
- Top objections (60-second versions):

| Objection | Pivot |
|-----------|-------|
| Accommodations cost too much | Many free; WOTC; JAN averages ~$300 |
| No time to train | On-site job coaching — ES trains on-site |
| Safety / liability | Safe job matching; optional WEX/OJT |
| Not hiring now | Free pipeline registration — 2 minutes |

- When stuck: “I will confirm with our team and follow up tomorrow.”
- After every conversation: confirm form submitted in Community Partners; log contact on participant records.

**Activity:** Pairs role-play [Employer outreach script](../scripts/employer-community-partners-outreach.md) — **10 minutes each** (trim third scenario if behind).

**Manual ref:** [Employer outreach script](../scripts/employer-community-partners-outreach.md)

## Session 3 — Reporting (40 min)

**Goal:** ES can open Joshua Tree Reports, complete an SE Monthly for a test client, and knows when to use other GA forms.

### 3A — Reports vs Pro (10 min)

**Talking points:**

- **Wayfinder Pro exports** = your CSV downloads for grants and analysis.
- **Joshua Tree Reports** = **official** GVRA PDFs → email + Google Drive. Different app, different sign-in session.
- Open from Pro: sidebar → **Reporting** (prefills client when launched from a profile).
- Direct URL: **wayfinder-reports.thejoshuatree.org** — `@thejoshuatree.org` only.

**Georgia report types (know the list; master SE Monthly first):**

| Report | When | Client? | Signature? |
|--------|------|---------|------------|
| **SE Monthly** | Monthly per active client | Yes | Yes |
| **VPR** | Vocational progress | No | No |
| **JTSG VMR** | Vocational monthly | Yes | Yes |
| **EVF** | Employment verification | No | No |
| **JTSG TSVS** | Vocational time sheet | No | No |

**Manual ref:** [Joshua Tree Reports Guide](../reports/joshua-tree-reports-guide.md) · [ES Quick Reference](../reports/joshua-tree-reports-es-quick-reference.md)

---

### 3B — SE Monthly hands-on (25 min)

**Talking points:**

1. Select **Georgia** → **SE Monthly**.
2. Select **client** from caseload.
3. Review **prefill** from contact logs (job development) — *why Session 1 contact logging matters*.
4. Complete remaining fields; **Employment Specialist name** editable if needed.
5. Capture **signature** when required.
6. Submit → PDF → email + Drive.

**Monthly checklist (every active GA client):**

- [ ] Correct client and reporting month
- [ ] Contact logs current in Pro **before** prefill
- [ ] Review recalled prior-month data
- [ ] Signature captured if required
- [ ] Confirm PDF in email / Drive

**Hands-on:** Walk through SE Monthly on a test client (or facilitator demo if sandbox limited).

**Troubleshooting (table only — do not dwell):**

| Issue | Action |
|-------|--------|
| Client not in dropdown | Confirm GA caseload in Pro |
| Prefill empty | Log contacts first; check month |
| Google / PDF error | WF- code → Bryan |
| Sign-in fails | Clear cookies; use Reports URL directly |
